Introduction

TripScript is a music visualization IDE built with Javascript, HTML5 Audio, and Canvas. I made it so I could experiment with music visualization without continuously recompiling the code, reloading the page, or restarting the music.

All of the busywork (setting up contexts, connecting audio nodes, etc.) is taken care of behind the scenes. The interesting parts (drawing on canvas in response to music) are done in visualization scripts that can be loaded dynamically and swapped in and out at run-time.

TripScript is purely a client-side application. There is no server-side code or database. The scripts are saved to your browser's local storage, but can be exported to a .js file for portability.

A visualization script is simply a block of javascript code that gets executed on every animation frame. The active script is treated as the body of a function that receives the following arguments:

ctx
2D rendering context for the canvas.
analyser
Web Audio API analyser node.
audioContext
Web Audio API audio context.
include
Function for loading other scripts.
isActive
true if the script is currently active; false if it was called via include

Analysing the Music

You can access audio data directly through the analyser and audioContext variables. However, I have included a music library that performs some common audio analysis tasks. Take a look at the included sample scripts for examples of how to use it. Use the wave and freq arrays to access audio data in the time and frequency domains, respectively. Each domain is divided into 256 discrete bands. Each array element is a value between 0 and 1, representing the magnitude of the signal in that band. For example: freq[0] will be close to 1 during a strong bass drum beat.

Drawing on Canvas

The ctx variable contains the drawing context for the canvas so you can stroke and fill to your heart's content. Take a look at the built-in scripts for some simple examples of what can be done. If this is your first time using canvas, check out Mozilla's Canvas Tutorial.

Including Library Scripts

Use the include() function to load a library script. It takes a single string argument: The name of the library to load. It returns an object containing all of the public variables & functions defined in that library. Includes should always be done at the top of your script and should not be placed inside functions, loops, or conditionals.

Creating Library Scripts

When a script is loaded from the include() function, it will be treated as a constructor and the constructed object will be returned. To expose your library's public functions, simply attach them to the execution context, this. Local variables will not be visible to the calling script.

Each library will only be constructed once. Subsequent calls to include() will simply return the existing object. This means that included scripts will only execute on the very first animation frame. If you want library code to execute on every frame, put it inside a public function and call that function from the active script.

TIP: You can also use this within the active script to share variables across animation frames.
